MCP検証 Google Sheets Google Drive 名寄せ

Google Sheets MCP 動作検証レポート

Claude Code から Google Drive / Sheets をMCP経由で操作し、架空データ2シートを作成・突合した際の操作記録と精度評価。2026年4月15日実施。

01実施内容・操作ステップ
1
Google Drive にフォルダ作成
マイドライブ直下に Claude Code フォルダを新規作成。MCP create_file（mimeType: folder）で実行。
2
「住所録」スプレッドシート作成（100行）
上記フォルダ内に 住所録 を作成。架空の氏名・郵便番号・都道府県・市区町村・番地・電話番号・メールアドレスを7列×101行（ヘッダー含む）で一括投入。
3
「取引先一覧」スプレッドシート作成（10行）
同フォルダ内に 取引先一覧 を作成。住所録から抜粋した10名に架空の銀行口座情報（銀行名・支店名・口座種別・口座番号）を付与。氏名表記は意図的に揺らせた（敬称・スペース・法人略称など）。
4
両シートを読み取り・名寄せ突合
2ファイルを sheets_read_values で取得し、表記揺れを無視して氏名突合。一致した行の住所録 H〜K列に口座情報を書き込み。

  • ✅処理できた揺れ：姓名間スペースの有無・全半角・末尾敬称（様/君）・括弧内付加情報・法人略称（株式会社→(株)）。いずれも正規化後の文字列比較で正確に突合。
  • 🔍除外の妥当性：「高橋 集介」vs「高橋 雄介」は異なる漢字を使った別人。表記揺れではなくデータ不整合であり、除外は正しい判断。ファジーマッチ（類似度スコア）を使えば誤マッチするリスクがある箇所。
  • ⚠️同姓同名リスク：今回は100件規模のため同姓同名なし。実運用では複数ヒットへの対処ロジックが必要。郵便番号・電話番号などの補助キーとの組み合わせ突合が推奨。
  • 🛠MCPの挙動：シート名がデフォルト「Sheet1」ではなく「シート1」だったため、初回の範囲指定でエラー発生。get_spreadsheet_info でシート名を事前確認するフローが必須。
注意：本検証で使用したデータ（氏名・住所・口座番号）はすべて架空です。実在の個人・法人・金融機関とは無関係です。
