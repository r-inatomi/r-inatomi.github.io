実証レポート Googleスプレッドシート Googleドライブ 名前照合

Claude がスプレッドシートを直接操作する
——何ができて、どこまで使えるか

プログラミングの知識なしで、会話するだけでGoogleドライブにファイルを作り、データを入れ、2つのファイルを照合できるか試した記録。2026年4月15日実施。

01今回試したこと
📁
Googleドライブにフォルダを作った
「Claude Code というフォルダをマイドライブ直下に作って」と指示するだけで、Googleドライブ上にフォルダが作成された。
📋
架空の住所録を100行作った
「架空の住所録を100行作って」と指示。氏名・郵便番号・都道府県・市区町村・番地・電話番号・メールアドレスの7列、100行分のデータが自動で入力された。
🏦
別ファイルに取引先＋口座情報を作った
同じフォルダに「取引先一覧」ファイルを別途作成。住所録の100人のうち10人を抽出し、架空の銀行口座情報（銀行名・支店名・口座種別・口座番号）を追加した。
🔗
2つのファイルを照合して、口座情報を住所録に転記した
「住所録と取引先一覧を突き合わせて、一致する人の口座情報を住所録に追記して」と指示。名前のスペースの有無や敬称（様・君）の違いも自動で読み飛ばして照合し、該当行に口座情報を書き込んだ。
02名前の表記が違っていても照合できたか

取引先一覧の名前は、意図的に表記を揺らして作成した（スペースなし・敬称あり・法人略称など）。Claudeがどこまで「同一人物」と判断できるか確認した。

03何がポイントか
💬
操作はすべて「会話」だけ
「住所録を100行作って」「2つのファイルを照合して転記して」——指示はすべて日本語の会話のみ。ファイルを手動で開く操作はゼロ。
🔄
2ファイルをまたいで操作
別々のスプレッドシートを読み込み、照合して、片方に書き戻す——複数ファイルにまたがる作業をワンステップで完了。
🧠
「ゆらぎ」を読み飛ばして照合
スペース・敬称・法人略称など、人間が目で合わせるような判断も自動で処理。「無視して突合して」という指示を正確に実行。
🚫
別人は別人として除外
「ゆらぎを無視」といっても、漢字が異なる別人まで無理にマッチさせることはなかった。判断の根拠も説明できる。
今回やっていないこと（拡張できること）：
既存ファイルの読み込み・更新 / 条件付き書式の設定 / フォームと連動した自動入力 / 定期的な自動実行（スケジュール実行） など。今回は「作る・照合する・転記する」の基本3操作を検証した。
04まとめ：どんな用途に使えるか
  • 📥 データ入力の代行：ひな形を渡して「100件分のダミーデータを入れて」「条件に合うデータを別シートに抜き出して」といった作業をそのまま依頼できる。
  • 🔍 名寄せ・突合：顧客名簿と請求データなど、別々に管理しているファイルを照合して転記する作業。人間が目視で行うと抜け漏れが出やすい作業を正確に処理できる。
  • 📂 ファイル整理：フォルダ作成・ファイル作成・移動など、Googleドライブの操作を会話で指示できる。「○○というフォルダを作ってこのファイルを入れて」で完結。
  • ⚠️ 注意点：100件・2ファイル規模では問題なく動作したが、大量データ（数千行〜）や複雑な照合ロジックが必要な場合は、事前に動作確認を推奨。
